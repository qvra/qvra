# qvra

`qvra` is where sharp software gains public motion.

This repository is the front door of the field.
Its job is simple:
- show what is alive
- route people fast
- reduce time to first contact
- make the field legible without slowing it down

## Start by intent

### Use something now
- [`run`](https://github.com/qvra/run) — immediate utility anchor
- [`launch`](https://github.com/qvra/launch) — fast entry kits
- [`kit`](https://github.com/qvra/kit) — starter foundations and reusable bases

### Compare, test, and measure
- [`bench`](https://github.com/qvra/bench) — reproducible comparison and technical pressure

### Explore live experiments
- [`lab`](https://github.com/qvra/lab) — unstable prototypes and forward edges

### See visible proof
- [`show`](https://github.com/qvra/show) — demos, examples, and direct visual routes

### Track what is moving
- [`pulse`](https://github.com/qvra/pulse) — heartbeat, updates, and live field rhythm

## Structural spine

These repositories keep the field coherent without overpowering the anchors:
- [`site`](https://github.com/qvra/site) — public web surface
- [`meta`](https://github.com/qvra/meta) — shared standards and repo patterns
- [`core`](https://github.com/qvra/core) — deeper shared machinery
- [`index`](https://github.com/qvra/index) — field map and route catalog
